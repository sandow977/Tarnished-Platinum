Use this workflow for adding species to the game.

1. Pick an ID strategy first.
- Add your new enum in [species.txt](pokeplatinum/generated/species.txt).
- Insert **before** `SPECIES_EGG` so existing real species IDs stay stable.

2. Create the species asset folder.
- Folder name must be enum name lowercased without `SPECIES_`.
- Copy an existing species folder and edit it

3. Required files inside that folder.
- `data.json` (stats/types/learnset/evo/pokedex fields)
- `sprite_data.json` (animation/y-offset/shadow)
- `male_front.png`, `male_back.png`, `female_front.png`, `female_back.png`
- matching `.png.key` files
- `normal.pal`, `shiny.pal`
- `icon.png`
- `footprint.png`
- `meson.build` (same pattern as existing species)
- Reference example folder: [res/pokemon/abra](pokeplatinum/res/pokemon/abra)

4. Set species gameplay data.
- Edit  `data.json` using an existing mon as template.
- This includes base stats, abilities, evolutions, moves, egg groups, dex text, etc.

5. Set sprite behavior.
- Edit  `sprite_data.json` for front/back animation and offsets.

6. Register forms
- Edit [form_registry.json](pokeplatinum/res/pokemon/form_registry.json) (only for alternate forms).

7. Add to Sinnoh Regional Dex
- Edit [sinnoh_pokedex.json](pokeplatinum/res/pokemon/sinnoh_pokedex.json).


#IGNORE THIS (This section is for storage)

8. Build (regenerates enums/tables/NARCs).
```bash
cd ~/pokeplatinum
make -j1
```

9. Validate generated outputs.
- `build/generated/species.h` should include your `SPECIES_FAKEMON`.
- `build/res/pokemon/*.narc` should rebuild without errors.

10. Add usage hooks for wild encounters, trainer parties, scripts

11. Test in-game.
- Spawn it (wild or trainer), check sprite/icon/cry/learnset/evolution/pokedex.
